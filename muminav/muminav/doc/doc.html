Server [Client Entwickler ]

Eigentlich ist es vorgesehen, dass das Applet eine Anfrage an ein Servlet stellt, welches die Daten aus einer Datenbank liest, entsprechend aufbereitet und an das Applet sendet.

Momentan exisitiert aber eine entsprechende Testumgebung noch nicht, so dass die XML Files noch per Hand geschrieben werden mussen.

Mit den Releases liefern wir auch eine Testumgebung mit (Verzeichnis html), die aus folgenden Dateien besteht:

*.xml Navigationsnetzdefinitionen
Applet.html enthalt das Applet
content*.html Inhaltseiten, die vom Applet aufgerufen werden.
frameset.html baut das Frameset aus Applet und Inhaltsseiten zusammen

Was wird fur die Installation benotigt:

  • Auf jeden Fall eine html- Datei, die das Applet einbindet.

    Dieses erhalt uber den Parameter XMLFile den Pfad zur XML- Datei mit der Netzdefinition

  • Die XML- Datei, die dem Applet als Parameter ubergeben wurde.

  • Ein Webserver, der das ganze bereitstellt. Es geht nicht ohne! Die Daten werden per http geladen!

    Die XML- Datei muss dabei auf dem gleichen Server liegen, von dem aus das Applet geladen wird.

    Die Konfiguration des Webservers ist den Anleitungen zu diesem zu entnehmen.

Die Installation

Zur Installation der Testumgebung kopiert man den Inhalt des Verzeichnisses html in das vom Webserver bereitgestellte Directory. Entweder man passt jetzt in der Datei Applet.html den Pfad zur Klasse muminav.Muminav.class an oder kopiert einfach den Inhalt des Verzeichnisses classes in den Ordner, in dem die HTML Dateien liegen.

Anforderungen an die XML- Datei:

  • Es sollten nur Elemente definiert werden, zu den unter dem skin Package auch Klassen existieren.
  • Das oberste Element muss NavNet heissen und als Attribute den Skin (skin="muminav.skin.math") und die Dimension des Netzes (width="35" height="78") ubergeben bekommen.
  • Texte zwischen den Elementen konnen definiert werden, finden aber keine Beachtung. Alle Parameter zum Zeichnen des Netzes werden den Elementen per Attribut ubergeben.

Detailliertere Angabe sind im Skin- Tutorial zu finden.
